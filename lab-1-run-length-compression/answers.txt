2. This compression works well when the file has many repeated patterns or characters, such as "AAAAAA" or long sequences of the same values.

This compression works poorly when the file has little or no repetition, such as "ABCDEFG", because it may even increase the file size instead of reducing it.

3. The file "1mil_strong_bias" compresses the best because it contains many repeated values (especially the number 0 appearing very frequently). More repetition leads to better compression.

4. Compression Rations (K=1)

The random file compresses poorly because it has almost no repetition. The weak bias file compresses better due to some repeated values. The strong bias file compresses the best because it has many repeated values, allowing long runs to be stored efficiently.
