6. What do you notice about the output ordering? Why do you think this is?
The output order changes every time the program is run. This happens because threads execute concurrently and the OS scheduler decides when each thread gets CPU time. Since scheduling is non-deterministic, there is no guarantee about which thread runs first, so the print order varies between runs.
